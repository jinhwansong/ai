# 📰 AI 기반 경제 뉴스 & 전략 브리핑 서비스

> 해외 경제 뉴스를 AI로 분석해  
> **한국 사용자 기준으로 ‘오늘의 경제 흐름 + 투자 판단 보조 인사이트’를 제공하는 브리핑 서비스**

이 프로젝트는 **해외 경제 뉴스 API(영문 기준)** 를 활용해  
글로벌 경제 뉴스를 수집하고,  
AI(GPT / Gemini)를 통해 **핵심 이슈 요약, 시장 영향 분석, 섹터·ETF 전략 가이드**를  
한국 사용자 관점에서 제공하는  
**프론트엔드 중심의 AI 경제 브리핑 서비스**입니다.

단순한 뉴스 번역이 아니라,  
**“오늘 시장을 어떻게 해석할 것인가”** 에 초점을 맞춘  
**개인 투자 판단 보조용 대시보드 UX**를 목표로 설계했습니다.

---

## 🎯 Problem

프로젝트를 시작하며 아래와 같은 현실적인 문제를 마주했습니다.

- 대부분의 해외 뉴스 API는 **영문 검색 기준**
- 한국어 키워드 검색 시 결과가 나오지 않는 문제
- 원문(영어) 위주의 뉴스 UI는  
  한국 사용자에게 높은 진입 장벽
- 뉴스는 많지만  
  **“그래서 오늘 시장에 무슨 영향이 있는지”** 바로 알기 어려움
- 검색 결과가 없을 경우 사용자 경험이 단절됨

👉 단순 정보 나열이 아닌  
👉 **경제 흐름을 ‘해석해서 전달하는 UX’** 가 필요하다고 판단했습니다.

---

## 💡 Solution

### 1. 한글 → 영어 → 한글 파이프라인 설계

- 한글 입력 → 뉴스 검색에 적합한 영어 키워드 변환
- 단순 번역이 아닌 **검색 정확도를 고려한 키워드 정제**
- 최종 결과는 항상 **한국어 요약 중심**으로 제공

---

### 2. AI 요약을 ‘브리핑’의 중심 콘텐츠로 재정의

- 기본 노출 콘텐츠: **AI 요약 리포트**
- 원문 기사: 확인용 옵션 (토글 / 버튼)
- 뉴스 사이트가 아닌  
  **경제 브리핑 서비스라는 정체성**을 UI에 반영

---

### 3. 매일 아침 9시, AI 기반 데일리 분석 리포트

- 매일 오전 9시 자동 실행
- 그날 수집된 글로벌 경제 뉴스를 기준으로
  - 핵심 이슈 요약
  - 글로벌 매크로 요약
  - 한국 증시(KOSPI/KOSDAQ) 영향 분석
  - 섹터별 영향도 재산정
  - ETF 전략 가이드 생성
- **하루 단위 스냅샷 리포트**로 저장 및 제공

---

### 4. 검색 실패(Empty State) UX 설계

- 검색 결과가 없을 경우 공백을 노출하지 않음
- 해외 뉴스 기준이라는 맥락을 명확히 안내
- 다음 행동을 유도하는 UX 제공
  - 다른 키워드 검색
  - 추천 키워드 제안

---

## ✨ 주요 기능

### 1. 데일리 경제 브리핑 (자동 생성)

- 글로벌 경제 뉴스 20~40개 수집
- AI가 **3~5개의 핵심 이슈로 요약**
- 글로벌 매크로 요약 (미국 / 중국 / 유럽)
- 한국 증시 관점 영향 분석
- 캐싱을 통한 중복 요청 방지

---

### 2. 섹터 영향도 분석 & 전략 가이드

- 주요 섹터별 영향도 점수화 (0~5)
  - 예: 반도체, 2차전지, 금융, 소비재 등
- 단순 추천이 아닌 **접근 전략 제안**
  - 분할 매수 / 관망 / 방어적 접근 등
- 전일 대비 변화가 있을 경우 강조 표시

---

### 3. ETF 전략 브리핑 (개인 투자 판단 보조)

- 섹터 분석 결과를 기반으로 ETF 전략 제안
- 예:
  - 반도체 → KODEX 반도체 / SOXX
  - 성장주 → QQQ / 나스닥100
  - 방어 전략 → 채권 ETF
- **“사라”가 아닌**
  - “어떤 상황에서 어떻게 접근할 수 있는지” 중심 설명

---

### 4. 검색 기반 뉴스 & 이슈 분석

- 사용자가 직접 키워드 입력
- 관련 뉴스 수집 후 AI 요약 제공
- 해당 이슈가
  - 글로벌 매크로
  - 한국 증시
  - 특정 섹터에 미치는 영향 분석

---

### 5. 관심 키워드 기반 개인화 브리핑

- 온보딩에서 관심 키워드 선택
- 예: `금리`, `AI`, `반도체`, `환율`, `유가`
- 선택 키워드 중심의 데일리 브리핑 제공

---

### 6. 온보딩 기반 개인화 (로그인 없음)

- 첫 방문 시 관심 키워드 선택
- localStorage 기반 저장
- 이후 메인 페이지에서 개인화 콘텐츠 제공

---

## 🧠 UX 설계 포인트

- **AI 요약과 분석을 항상 최우선 정보로 배치**
- 원문 기사는 참고용 옵션으로 제공
- 뉴스 → 해석 → 전략 흐름이 자연스럽게 이어지는 구조
- “오늘은 관망” 같은 판단도 허용하는 톤
- 검색 결과 없음 상태에서도  
  사용자가 다음 행동을 즉시 선택할 수 있도록 설계
- 원페이지 구조 + 섹션 앵커 기반 탐색
